# Contributing to Resource Scarcity Research

Thank you for your interest in contributing! This is an open research project, and every contribution — from fixing a typo to adding a new country comparison — makes this work more valuable.

---

## 📋 Before You Contribute

1. **Read the research first** — at least the [README](README.md) and the [Executive Brief](research1.md#s1-executive-brief--1-page-for-decision-makers)
2. **Check existing content** — make sure your contribution isn't already covered
3. **Use credible sources** — government reports, RBI data, international agencies (IEA, FAO, WHO, World Bank), peer-reviewed journals, or reputable news outlets

---

## 🔧 How to Contribute

### Option 1: GitHub Pull Request (Preferred)

1. Fork this repository
2. Create a new branch (`git checkout -b add-brazil-comparison`)
3. Make your changes
4. Commit with a clear message (`git commit -m "Add Brazil's drought management as country comparison #9"`)
5. Push to your fork (`git push origin add-brazil-comparison`)
6. Open a Pull Request with a description of what you added and why

### Option 2: Open an Issue

If you don't want to edit files directly, [open an Issue](../../issues) with:
- **Title:** Clear description (e.g., "DAP import percentage needs updating")
- **Body:** What needs to change, and the correct information with a source link

### Option 3: Email / Direct Contact

If you're not on GitHub, you can reach out via the contact information in the repository.

---

## 📝 Contribution Guidelines

### Adding Case Studies
- Include: **What happened, When, Where, Impact on India, Source**
- Use the same format as existing case studies in `research.md`
- Minimum 1 credible source per case study

### Adding Country Comparisons
Follow this structure:
```
## X. COUNTRY NAME — Topic

The Challenge [Country] Solved: [1-line summary]

• What [Country] Does:
  o [Policy/programme 1]: Description
  o [Policy/programme 2]: Description

• How India Can Adapt:
  o [Specific action 1]
  o [Specific action 2]
```

### Updating Data
- Always include the **source URL** and **date** of the data
- If updating a number, note both the **old value** and **new value** in your PR description

### Adding Translations
- Place translations in a `/translations/` folder
- Name format: `executive_brief_[language].md` (e.g., `executive_brief_hindi.md`)
- Translate only the Executive Brief (Section S1) first — it's the most impactful section

---

## ⚠️ What We Don't Accept

- ❌ Political opinions or party-specific commentary
- ❌ Unverified claims without credible sources
- ❌ Content that promotes any commercial product or service
- ❌ Copy-pasted content from copyrighted sources (paraphrase and cite instead)

## 🏷️ Issue Labels

| Label | Description |
|-------|-------------|
| `data-update` | Updating statistics, prices, or percentages |
| `new-case-study` | Adding a new shortage case study |
| `country-comparison` | Adding a new country's approach |
| `fact-check` | Verifying or correcting existing claims |
| `translation` | Translating content into Indian languages |
| `enhancement` | Improving existing analysis or adding depth |
| `bug` | Fixing errors in data, grammar, or formatting |
